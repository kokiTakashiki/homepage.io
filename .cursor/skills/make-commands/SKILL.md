# Make Commands

このプロジェクトでは、開発ワークフローを効率化するためにMakefileを使用しています。

## 利用可能なコマンド

### make setup

開発に必要なツールをインストールします。

- Prettier (コードフォーマッター)
- html-validate (HTML検証ツール)

**使用タイミング:**

- プロジェクトの初回セットアップ時
- 依存パッケージを更新したい時
- ユーザーが「セットアップ」「インストール」などと言及した時

**実行方法:**

```bash
make setup
```

### make format

プロジェクト全体のコードをPrettierでフォーマットします。

**使用タイミング:**

- コードのフォーマットを統一したい時
- コミット前にコードを整形したい時
- ユーザーが「フォーマット」「整形」などと言及した時

**実行方法:**

```bash
make format
```

### make validate

HTMLファイルをエラーのみチェックします（警告は無視）。

**使用タイミング:**

- HTMLの文法エラーをチェックしたい時
- 重大なエラーだけを確認したい時
- ユーザーが「検証」「バリデーション」「チェック」などと言及した時

**実行方法:**

```bash
make validate
```

### make validate-all

HTMLファイルを警告含めて全てチェックします。

**使用タイミング:**

- HTMLの詳細な問題点を全て確認したい時
- コードの品質を徹底的にチェックしたい時
- ユーザーが「詳細チェック」「全てチェック」などと言及した時

**実行方法:**

```bash
make validate-all
```

### make help

利用可能なコマンドの一覧を表示します。

**実行方法:**

```bash
make help
```

## 検証ツールについて

このプロジェクトでは `html-validate` を使用しています。

- 純粋なJavaScript製（Javaランタイム不要）
- オフラインで動作
- HTML5標準に準拠した検証

設定ファイル: `.htmlvalidate.json`

## 注意事項

- `make setup` を実行する際は、npm がインストールされている必要があります
- コマンドはプロジェクトのルートディレクトリで実行してください
- エラーが発生した場合は、エラーメッセージを確認して対処してください
